## 调研要求
1. 强制 WebSearch，禁止用训练数据填充；如果当前运行环境没有搜索能力，要明确说“无法联网搜索”，不要假装查过。
2. 先判断调研对象类型，再选择信源。真正要迁移的技能是“找到这个领域的高质量证据栖息地”，不是照搬某个领域的固定平台清单；不要把某一类项目的搜索模板套到所有任务上。
   - AI / 软件 / 开发者工具 / 产品公司：官网、文档、定价、GitHub、Product Hunt、Reddit、36Kr、少数派、量子位、开发者社区、用户评论。这些平台在产品调研中很有价值，因为它们承载了真实使用、开发活跃度、发布反馈、定价与社区讨论。
   - 学术 / 社科 / 人类学 / 设计研究 / 艺术展览 / 公共文化项目：官网、研究机构/大学页面、论文/博士论文、项目报告、展览图录、策展说明、讲座/访谈、媒体报道、参与者原声、学术引用。这里的“高质量信源”通常不在产品平台里；除非项目本身是软件/开源工具，不要默认搜索 Product Hunt、GitHub、量子位这类产品/开发者平台。
   - 书籍 / 课程 / 教材：出版社官网、作者主页、图书馆/WorldCat、Google Books、豆瓣/京东/当当/出版社中文站、课程官网。
   - 活动 / 会议 / 营地 / 社群：官网、报名页、往届总结、议程、公开视频、志愿者/参与者说明、签证/地点/票务官方信息、当地媒体。
3. 每个关键数据点标注：来源 + 日期 + 置信度；不要用“WebSearch”作为来源名，要写清楚具体页面或机构。
4. 如果用户问“影响、反馈、价值、成果”，先按对象类型翻译成合适问题：
